简单工厂模式说明：
    给定的条件不同，生成的类不同。例如：给"zhaji"，生成ZhaJi类，给"shutiao"，生成ShuTiao类。
    优点：1.解耦
         2.可以自判断生成什么类，使用配置文件可以更改生成的类
    缺点：1.工厂类出问题全出问题
         2.类的个数增多

实现思路：
    根据一种类型抽象出接口或者抽象类：IFood。
    对这个抽象出的概念提供不同的实现：ShuTiao,ZhaJi。
    提供工厂，根据不同的条件生成不同的类：生成ShuTiao或者ZhaJi用IFood统一接收。在工厂类中使用简单的if..else或者switch判断即可。

应用场景：JDBC根据数据库的不同生成不同的Statement
